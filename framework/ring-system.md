# Ring System

Das Ring-System ist eine der Kernmechaniken des Challenger Build Systems.

Ein Ring ist immer eine Ebene im Bewertungskontext.

Jeder weitere Ring geht eine Ebene tiefer und macht die Bewertung spezifischer.

---

## Core Rule

```text
Ein Ring = eine bewertbare Ebene
Jeder innere Ring = eine spezifischere Ebene
```

Jede Ebene, die man sinnvoll bewerten kann, darf als eigener Ring existieren.

---

## Warum Ringe wichtig sind

Eine Leistung kann nie isoliert bewertet werden.

Beispiele:

- Eine 2 in Mathematik ist nicht überall gleich viel wert.
- Master auf EUW ist nicht gleich Master auf jeder Region.
- Top 1 % in einer starken Population ist anders als Top 1 % in einer schwachen Population.

Deshalb braucht jede Leistung ihren Kontext.

---

## Ring-Logik

```text
Ring 1 = größte Region / Population
Ring 2 = konkreter Bereich / Queue / System
Ring 3 = Spezialisierung
Ring 4 = konkrete Leistung
Ring 5+ = weitere Details, falls relevant
```

Jeder Ring besitzt:

- eigene Population
- eigene Difficulty
- eigene Seltenheit
- eigenes Ranking
- eigenen Percentile-Wert

---

## Jeder Ring kann separat bewertet werden

Nicht nur die finale Leistung wird bewertet.

Jeder einzelne Ring kann ein eigenes Ranking haben.

Beispiel:

```text
Europa
→ HTL IT
→ Matura
→ Wahlfach Künstliche Intelligenz
→ Note 2
```

Dabei kann man separat bewerten:

1. Wie stark/schwierig ist HTL IT im Vergleich zu anderen Schulformen?
2. Wie selten ist es, Matura in diesem Kontext zu absolvieren?
3. Wie selten ist das Wahlfach Künstliche Intelligenz?
4. Wie gut ist eine 2 innerhalb dieses Fachs?

---

## Beispiel: Education Build

### Ring 1

```text
Europa
```

Allgemeine Region.

---

### Ring 2

```text
HTL IT
```

Bewertbare Ebene.

Fragen:

- Wie viele Menschen wählen HTL IT?
- Wie viele schließen HTL IT erfolgreich ab?
- Wie schwer ist HTL IT im Vergleich zu anderen Schulformen?

---

### Ring 3

```text
Matura
```

Bewertbare Ebene.

Fragen:

- Wie viele erreichen die Matura?
- Wie viele brechen vorher ab?
- Wie hoch ist die Abschlussquote?

---

### Ring 4

```text
Wahlfach Künstliche Intelligenz
```

Bewertbare Ebene.

Fragen:

- Wie viele wählen dieses Fach?
- Wie spezialisiert ist es?
- Wie schwer ist es relativ zu anderen Wahlfächern?

---

### Ring 5

```text
Note 2 in Künstlicher Intelligenz
```

Konkrete Performance-Ebene.

Fragen:

- Wie viele schaffen eine 2 oder besser?
- Wie viele bestehen nur knapp?
- Wie verteilt sich die Notenpopulation?

---

## Beispiel: League of Legends

```text
Europa
→ EUW
→ SoloQ
→ Rolle
→ Rank / LP
```

Jeder Ring ist bewertbar:

- EUW hat eine bestimmte Server-Difficulty.
- SoloQ ist anders als FlexQ.
- Jede Rolle hat andere Schwierigkeit und Population.
- Rank / LP zeigt die konkrete Performance.

---

## Beispiel: Fitness

```text
Europa
→ Natural Bodybuilding
→ Männer
→ Altersklasse 18–25
→ Körperfett / Muskelmasse / Kraftwerte
```

Jeder Ring gibt Kontext.

Eine Leistung wird dadurch nicht nur absolut, sondern relativ zur Population bewertet.

---

## Wichtiges Prinzip

Das System fragt nie zuerst:

```text
Wie gut ist die Person?
```

Sondern:

```text
1. In welcher Region spielt sie?
2. In welchem System / Build spielt sie?
3. Welche Spezialisierung liegt vor?
4. Welche konkrete Performance wurde erreicht?
5. Wie selten ist diese Performance innerhalb aller Ringe?
```

---

## Ziel

Das Ring-System macht Leistung vergleichbarer, weil es Kontext, Difficulty und Population berücksichtigt.

Dadurch wird aus einer isolierten Aussage wie:

```text
Ich hatte eine 2 in Mathematik.
```

Eine strukturierte Aussage wie:

```text
Europa → HTL IT → Matura → Mathematik → Note 2
```

Erst danach kann ein Percentile- oder Rank-Wert berechnet werden.
